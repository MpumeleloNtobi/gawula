# Concept Cards

Use these only after open discussion. Show one concept at a time.

## Concept 1: Complete Your Meal

After you add a main meal, the app suggests useful add-ons from nearby same-hub stores: drinks, dessert, sides, snacks, or essentials. You can add them to the same basket before checkout.

Customer promise:

One order that covers the whole meal.

Questions:

1. What does this sound like to you?
2. When would it help?
3. Which add-ons would you actually use?
4. Would this feel helpful or like upselling?
5. What should the app show before you add the item?

## Concept 2: Everyone Gets What They Want

When ordering for a group, each person can choose items from different same-hub stores. The organizer checks out once and the order arrives together where possible.

Customer promise:

No one has to compromise on one restaurant.

Questions:

1. Who would you use this with?
2. Would this make group ordering easier?
3. What would you worry about?
4. Who should pay?
5. Should people be able to set budgets or deadlines?

## Concept 3: One Basket Across Stores

You can add items from selected stores in the same food court or hub to one basket, see each store clearly, then pay once.

Customer promise:

Different stores, one checkout.

Questions:

1. When would this be better than placing separate orders?
2. How many stores would feel manageable?
3. Would you expect one delivery fee or multiple fees?
4. What if one store is delayed?
5. Would you use this for delivery, pickup, or both?

## Concept 4: Protected Delivery

The app shows clear store-by-store order status, stronger delivery proof, and an easy way to report a missing item, wrong item, or not-received order.

Customer promise:

If something goes wrong, it is clear who fixes it.

Questions:

1. Does this solve a real problem for you?
2. What proof would make you trust delivery?
3. What should happen when one item is missing?
4. What refund timeline feels fair?
5. Would this matter more than extra ordering features?
